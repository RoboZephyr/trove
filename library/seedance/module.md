# Doubao Seedance 2.0 Usage Guide

## ⚠️ Critical Constraints (read before writing code)

1. **Account funding gate: balance ≥ ¥200 OR an active resource package** — without one of these, you cannot enable Seedance 2.0 (full or Fast). The first call returns a model-not-activated style error, not a billing one. Top up at https://console.volcengine.com/ark before integrating.
2. **Five task statuses, not the usual three** — `queued` → `running` → `succeeded` / `failed` / **`expired`** (the last one is auto-set when a queued/running task exceeds `execution_expires_after`, default 48h). Polling code that only checks `succeeded`/`failed` will hang forever on stuck tasks.
3. **Result video URL expires in 24 hours** — the `video_url` returned is a presigned TOS URL with `X-Tos-Expires=86400`. If you need durable storage, download immediately, or use Volcengine TOS data subscription to auto-archive into your own bucket. **Do not** hand the raw URL to end users for any flow longer than a few hours.
4. **Three input scenarios are mutually exclusive** — first-frame I2V (`role: first_frame`) / first-and-last-frame I2V (`role: first_frame` + `role: last_frame`) / multimodal-reference (`role: reference_image|video|audio`). You **cannot** mix `first_frame` with `reference_image` in the same task. Multimodal-ref can SIMULATE first/last-frame via the prompt ("首帧为图片1, 尾帧为图片2"), but if you need strict pinning, use the dedicated first+last-frame mode.
5. **No real-person faces in input references** — uploading reference images / videos containing real human faces is blocked by content moderation. Three official escape hatches:
   - Reuse a face that was itself generated by Seedance 2.0 / 2.0 Fast / Seedream 5.0 lite **within the last 30 days, same account, original file** (re-encoding breaks trust) — pass it back via a `tos-cn-beijing.volces.com` URL.
   - Use a preset virtual portrait from the 虚拟人像库. Pass `asset://<asset-id>` instead of a `url`. Free + compliant + skips face moderation.
   - Onboard an authorized real-person asset via the platform's identity-verification flow. Same `asset://<asset-id>` shape after enrollment.
6. **Prompt references must be `<modality-type> n`, not asset IDs** — when you pass two `reference_image`s, the prompt refers to them as "图片1" / "图片2" (or "image 1" / "image 2"), indexed by their order within the same `type` in the `content` array. `asset://...` is ONLY for the URL field; **never** put the asset ID into the prompt text.
7. **`duration` is an integer in seconds, range 4–15** — number, not string. Outside this range → 400. Different from providers that take strings like `"5"`.
8. **Model ID format is `doubao-seedance-2-0-260128` (NOT `seedance-2.0` or any hand-typed friendly name)** — the trailing `-260128` is the version pin. An Endpoint ID (`ep-xxxx`) you provisioned in the console also works. Don't try `Seedance 2.0` / `doubao-seedance-2.0` / `seedance-pro` — those are 400s.
9. **`generate_audio` defaults to `true` for Seedance 2.0** — output will have synced audio unless you explicitly set `false`. Set to `false` for silent video if you intend to dub yourself, otherwise you'll be confused why your "T2V" output has voices.
10. **`ratio: "adaptive"` is the fix for the most common visual bug** — first-frame / first-and-last-frame tasks where the input image aspect ratio doesn't match the requested `ratio` cause frame stretch / frame-jump artifacts. Either crop the input to the target ratio, or set `ratio: "adaptive"` to use the image's native ratio.
11. **Seedance 2.0 does NOT support offline inference (`service_tier: "flex"`)** — flex (the 50%-price queue tier) is Seedance 1.5 pro only. If you pass `service_tier: "flex"` on Seedance 2.0, the parameter is ignored or errors. For batch jobs on 2.0, the only knob is `execution_expires_after`.
12. **Rate limits differ enterprise vs personal** — personal accounts: 180 RPM, 3 concurrent. Enterprise: 600 RPM, 10 concurrent. The concurrency limit is the tighter gate for batch workloads.
13. **`text + audio` and `audio-only` are NOT valid input combinations** — audio always needs at least one image or video alongside. Surfaces as a generic 400, not as "missing modality."
14. **Task IDs are only retained for 7 days from `created_at`** — after that the query endpoint returns 404 / task-not-found. If you might need to revisit a task (analytics, debugging, billing reconciliation), persist the create response immediately, including the `id` and your downloaded `video_url` content.
15. **Seedance 2.0 does NOT support `frames` or `camera_fixed`** — `frames` (sub-second duration control via frame count) and `camera_fixed` (lock camera to fixed shot) are Seedance 1.0 / 1.5 only. On 2.0, use `duration` (integer seconds) for length and prompt phrasing ("fixed camera", "static shot") for camera control.

## Auth

Single header, single secret. The Ark SDK reads `ARK_API_KEY` from the environment; for raw HTTP / OpenAI SDK, send it as a bearer token.

```bash
# Trove pattern — pull the key on demand, don't pre-export
ARK_API_KEY=$(jq -r .ARK_API_KEY ~/.trove/seedance/credentials.json)
```

```http
Authorization: Bearer <ARK_API_KEY>
Content-Type: application/json
```

Base URL: `https://ark.cn-beijing.volces.com/api/v3` (cn-beijing is the only region that ships Seedance today).

## Quickstart: the official Ark Python SDK

The documented primary path. Install:

```bash
pip install 'volcengine-python-sdk[ark]'
```

Then:

```python
import os, time
from volcenginesdkarkruntime import Ark

# Pull from Trove on demand (or rely on env var ARK_API_KEY)
api_key = os.environ.get("ARK_API_KEY")
client = Ark(base_url="https://ark.cn-beijing.volces.com/api/v3", api_key=api_key)

# 1. Submit
create_result = client.content_generation.tasks.create(
    model="doubao-seedance-2-0-260128",          # or doubao-seedance-2-0-fast-260128
    content=[
        {"type": "text", "text": "A cat walks across a snowy rooftop at golden hour"},
    ],
    ratio="16:9",
    duration=5,
    generate_audio=False,                          # silent (default is True on Seedance 2.0!)
    watermark=False,
)
task_id = create_result.id
print("submitted:", task_id)

# 2. Poll — handle all 5 statuses
TERMINAL = {"succeeded", "failed", "expired"}
while True:
    r = client.content_generation.tasks.get(task_id=task_id)
    if r.status in TERMINAL:
        break
    print(f"status={r.status}, retry in 30s")
    time.sleep(30)

if r.status == "succeeded":
    print("video URL (valid 24h):", r.content.video_url)
    if r.content.last_frame_url:
        print("last frame:", r.content.last_frame_url)
else:
    raise RuntimeError(f"seedance {r.status}: {r.error}")
```

Status values: `queued` → `running` → `succeeded` / `failed` / `expired`.

## Multimodal content shape

Every request body has the same backbone — a `content` array of typed items. The number and role of each item determines what kind of generation you're doing.

```python
content=[
    # Required: a text prompt (the only modality that can stand alone)
    {"type": "text", "text": "<your prompt>"},

    # Optional: 0–9 reference images, role tags the function
    {"type": "image_url", "image_url": {"url": "https://..."}, "role": "reference_image"},
    {"type": "image_url", "image_url": {"url": "https://..."}, "role": "first_frame"},
    {"type": "image_url", "image_url": {"url": "https://..."}, "role": "last_frame"},

    # Optional: 0–3 reference videos
    {"type": "video_url", "video_url": {"url": "https://..."}, "role": "reference_video"},

    # Optional: 0–3 reference audios
    {"type": "audio_url", "audio_url": {"url": "https://..."}, "role": "reference_audio"},
]
```

**Reference URLs must be publicly fetchable** — the model server pulls the content from your URL. TOS public-read buckets are the recommended host; arbitrary URLs (S3, R2, Cloudflare R2) work if their ACL allows anonymous GET. For local files, base64 is supported but discouraged when the request body would exceed 64 MB.

**Role tags**:
- `reference_image` — style / object / character continuity reference
- `first_frame` — pin this exact image as frame 0 of the output
- `last_frame` — pin this exact image as the final frame
- `reference_video` — copy motion / camera / style / subject from this video
- `reference_audio` — copy voice / timbre / BGM rhythm from this audio

**Valid combinations** (per official docs — anything else is rejected):
- text alone
- text (optional) + image
- text (optional) + video
- text (optional) + image + audio
- text (optional) + image + video
- text (optional) + video + audio
- text (optional) + image + video + audio

`text + audio` and `audio-only` are NOT valid.

## Input asset limits

### Images
- Format: jpg / png / webp / bmp / gif / tiff / heic / heif (heic/heif Seedance 2.0 only)
- Aspect ratio (W/H): in `(0.4, 2.5)`
- Each side: in `(300, 6000)` px
- Size per image: < 30 MB
- Request body total: ≤ 64 MB (do NOT base64-encode large files)
- Quantity: first-frame mode = 1, first+last-frame mode = 2, multimodal-ref mode = 1–9

### Videos
- Format: mp4, mov
- Resolution: 480p, 720p, 1080p
- Duration per video: `[2, 15]` s; up to 3 references; **total of all reference videos ≤ 15 s**
- Aspect ratio (W/H): `[0.4, 2.5]`
- Each side: `[300, 6000]` px
- Total pixels W×H: `[409600, 2086876]` (i.e. ~640² floor, ~1920×1080 ceiling)
- Size per video: ≤ 50 MB
- FPS: `[24, 60]`

### Audios
- Format: wav, mp3
- Duration per audio: `[2, 15]` s; up to 3 references; **total of all reference audios ≤ 15 s**
- Size per audio: ≤ 15 MB

## Top-level options (request body, alongside `content`)

| field | values | meaning |
|---|---|---|
| `model` | `doubao-seedance-2-0-260128` / `doubao-seedance-2-0-fast-260128` / `ep-<endpoint id>` | full picks quality, fast picks speed/cost; endpoint ID overrides both |
| `ratio` | `21:9` / `16:9` / `4:3` / `1:1` / `3:4` / `9:16` / `adaptive` | output aspect ratio. **Default `adaptive` on Seedance 2.0** (model picks based on prompt for T2V or the input image/video for I2V/multimodal). Override only if you need a fixed ratio |
| `duration` | int `4`–`15`, or `-1` | seconds. Default `5`. Number, NOT string. **`-1` = let the model pick** within the valid range (actual value returned in response) — useful for "longest-the-prompt-justifies" outputs, but be mindful it affects billing |
| `resolution` | `480p` / `720p` / `1080p` | Fast caps at 720p. Default `720p` |
| `watermark` | bool | Doubao "AI 生成" mark on lower-right. **Default `false`** |
| `generate_audio` | bool | **default `true`** for Seedance 2.0. When `true`, output mp4 has synced mono audio (voice + BGM derived from prompt or audio reference). Put dialogue in double quotes (`"hello"`) for cleaner voice synthesis. Set explicitly to `false` for silent video |
| `service_tier` | `default` | online inference only on Seedance 2.0. (`flex` is 1.5 pro only) |
| `execution_expires_after` | int seconds in `[3600, 259200]` | task timeout. Default `172800` (48h). After this, task moves to `expired` status |
| `return_last_frame` | bool | when `true`, response `content.last_frame_url` returns the tail frame as PNG (same W×H as the video, no watermark). Default `false`. The standard pattern for "chain videos with consistent character" — use this tail frame as the next call's `first_frame` |
| `callback_url` | string | webhook target for status changes. Ark POSTs the full query-task response body to this URL when status flips. 5s timeout, 3 retries on failure |
| `safety_identifier` | string ≤64 chars | hashed end-user ID; helps Volcengine detect policy abuse on your app. Pass a hash of the user's account, not the raw username/email |
| `tools` | `[{"type": "web_search"}]` | text-only inputs only. Lets the model search the web before generating |
| `seed` | int in `[-1, 2^32-1]` | reproducibility seed. Same request + same seed → similar (not guaranteed identical) output. `-1` (or omit) → random |

**Seedance 2.0 does NOT accept these fields** (silently ignored or 400):
- `frames` — use `duration` instead (integer seconds; sub-second control is 1.0/1.5 only)
- `camera_fixed` — phrase it in the prompt ("fixed camera, static shot")
- `draft` — Seedance 1.5 pro feature only

**Parameter passing — strict (preferred) vs loose**: pass these as top-level fields in the request body (strict validation, errors on bad values). Old loose form `--ratio 16:9 --duration 5` appended to the prompt text still works but silently ignores typos. Use strict.

## Use case recipes

### 1. Pure text-to-video (silent)

```python
content=[{"type": "text", "text": "Macro shot of a hummingbird drinking from a red flower, slow motion"}]
# generate_audio=False, ratio="16:9", duration=5, watermark=False
```

### 2. Image-to-video with first/last frame pinning

```python
content=[
    {"type": "text", "text": "Camera slowly pans left to right, golden hour"},
    {"type": "image_url", "image_url": {"url": "https://.../start.jpg"}, "role": "first_frame"},
    {"type": "image_url", "image_url": {"url": "https://.../end.jpg"}, "role": "last_frame"},
]
# Set ratio="adaptive" if your two images have different ratios
```

### 3. Multimodal — style image + motion video + voice audio

```python
content=[
    {"type": "text", "text": 'Promotional clip in the style of 图片1, with the motion of 视频1. Speaker says: "Try our new product."'},
    {"type": "image_url", "image_url": {"url": "https://.../style.jpg"}, "role": "reference_image"},
    {"type": "video_url", "video_url": {"url": "https://.../motion.mp4"}, "role": "reference_video"},
    {"type": "audio_url", "audio_url": {"url": "https://.../voice.mp3"}, "role": "reference_audio"},
]
# generate_audio=True for the voice to appear in output
```

### 4. Video edit (replace an object inside an existing video)

```python
content=[
    {"type": "text", "text": "Replace the perfume bottle in 视频1 with the cream from 图片1, keep camera motion identical"},
    {"type": "image_url", "image_url": {"url": "https://.../cream.jpg"}, "role": "reference_image"},
    {"type": "video_url", "video_url": {"url": "https://.../perfume_box.mp4"}, "role": "reference_video"},
]
```

### 5. Video extension / stitching (2–3 clips → one continuous video)

```python
content=[
    {"type": "text", "text": "视频1 ends at the doorway, then 视频2 enters the room, then 视频3 zooms into the painting"},
    {"type": "video_url", "video_url": {"url": "https://.../clip1.mp4"}, "role": "reference_video"},
    {"type": "video_url", "video_url": {"url": "https://.../clip2.mp4"}, "role": "reference_video"},
    {"type": "video_url", "video_url": {"url": "https://.../clip3.mp4"}, "role": "reference_video"},

```

### 6. Web-search-augmented text-to-video

```python
content=[{"type": "text", "text": "A photorealistic clip of the new Tesla Cybertruck pickup configuration"}]

# After completion: r.usage.tool_usage.web_search → search count (0 = no search performed)
```

## Driving the API from Node / curl (OpenAI-compatible)

The Ark surface accepts the OpenAI client SDK pointed at its base URL. For Node:

```typescript
import OpenAI from "openai";

const ark = new OpenAI({
  baseURL: "https://ark.cn-beijing.volces.com/api/v3",
  apiKey: process.env.ARK_API_KEY,
});

// The OpenAI SDK doesn't model Ark's `content_generation.tasks` namespace,
// so for video tasks specifically, hit the endpoint directly:
const r = await fetch("https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.ARK_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "doubao-seedance-2-0-260128",
    content: [{ type: "text", text: "A cat in a snowy rooftop, golden hour" }],
    ratio: "16:9",
    duration: 5,
    generate_audio: false,
  }),
});
const { id: taskId } = await r.json();

// Poll
while (true) {
  const g = await fetch(`https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks/${taskId}`, {
    headers: { Authorization: `Bearer ${process.env.ARK_API_KEY}` },
  }).then((r) => r.json());
  if (g.status === "succeeded") { console.log(g.content.video_url); break; }
  if (g.status === "failed" || g.status === "expired") throw new Error(g.error ?? g.status);
  await new Promise((r) => setTimeout(r, 30_000));
}
```

curl equivalent for create:

```bash
curl -X POST https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks \
  -H "Authorization: Bearer $ARK_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seedance-2-0-260128",
    "content": [{"type":"text","text":"A cat on a snowy rooftop, golden hour"}],
    "ratio": "16:9",
    "duration": 5,
    "generate_audio": false
  }'
```

## Using virtual / authorized portraits via `asset://`

For human-face content, the URL field accepts `asset://<asset-id>` in place of a public URL. The asset ID comes from:

- the 虚拟人像库 (preset compliant portraits, free) — browse + pick at https://console.volcengine.com/ark
- your own real-person asset uploaded via the identity-verification flow

```python
content=[
    {"type": "text", "text": "图片1 中美妆博主 introduces 图片2 的面霜 to the camera"},
    {"type": "image_url", "image_url": {"url": "asset://asset-20260401123823-6d4x2"}, "role": "reference_image"},
    {"type": "image_url", "image_url": {"url": "https://.../cream.jpg"}, "role": "reference_image"},
]
```

In the prompt, still call them "图片1" / "图片2" (positional within `image_url` items). **Never** drop the asset ID into the prompt text.

## Picking between Seedance 2.0 full vs Fast

| use case | pick |
|---|---|
| Max quality, willing to pay & wait | `doubao-seedance-2-0-260128` |
| Iteration / prototype / batch | `doubao-seedance-2-0-fast-260128` |
| 1080p output required | full only (Fast caps at 720p) |
| Multimodal / asset:// / web search | both support identically |

Both models accept all the same modality combinations, all the same role tags, and the same async pattern — they only differ in latency, output ceiling, and unit price.

## Prompt language + length

- All Seedance models accept Chinese and English prompts.
- Seedance 2.0 + 2.0 Fast additionally support **Japanese / Indonesian / Spanish / Portuguese**.
- Recommended length: **Chinese ≤ 500 chars**, **English ≤ 1000 words**. Beyond that, the model starts dropping details — pick the most important visual beats and lead with those.
- For audio output (`generate_audio: true`): wrap intended dialogue in double quotes — e.g. `she says: "I'll try the new cream"` — to get cleaner voice synthesis. Without quotes, the model may render the dialogue as descriptive text rather than spoken audio.

## Trusted-result chaining (keeping a face consistent across clips)

A face that Seedance 2.0 / Fast generated, or that Seedream 5.0 lite generated as an image, is whitelisted as a reference back into Seedance 2.0 for 30 days, same account, original file. Use this when you need character continuity across multiple clips:

1. First task: text-only → get a video → grab the last frame (set `return_last_frame: true`) OR get a Seedream image
2. Upload that frame / image to a public URL (TOS recommended)
3. Pass it as a `reference_image` to the next Seedance call

**Avoid re-encoding** (re-saving via a different tool, screen-grab, ffmpeg re-mux) — that breaks the trust signal. Use the original file straight from Volcengine's TOS bucket, or transfer it byte-for-byte to your own bucket.

---

## Polling vs callback

For interactive UX (a user is waiting), polling every 30s is the documented pattern. For batch / pipeline / serverless use:

- **`callback_url`** — pass a public HTTPS URL when creating the task. Ark POSTs the full query-task response body (same shape as `GET /tasks/{id}`) to that URL on every status change. 5s timeout, 3 retries.
- **Verify the source** — anyone can POST to your endpoint. Sign the callback URL with a long random token in a query string (`?token=...`), or use a secret-bearing path, then reject unauthenticated POSTs.

## Pricing pitfalls

- **Token-billed, not task-count** — the response `usage.completion_tokens` is the cost meter. Observed real numbers: **5s 480p text-only no-audio ≈ 50.6k tokens** (this module's first smoke). **11s 720p with audio ≈ 411k tokens** (per official doc example). Cost scales with duration × resolution × audio-on, sub-linearly past 5s.
- **Resource packages vs PAYG**: the `≥¥200 balance` gate is the entry threshold for PAYG. Resource packages (predeclared quotas at discount) are a separate SKU at https://console.volcengine.com/ark → 资源管理 → 资源包.
- **Fast model is cheaper but not strictly proportional** — typically Fast is ~40% the cost of full at the same duration/resolution, not 50/50.
- **`duration: 15` is NOT 3× the cost of `duration: 5`** — cost scales sub-linearly past 5s on most plans. Verify in your billing dashboard before assuming.
- **`service_tier: "flex"` does NOT apply to Seedance 2.0** — the 50% discount queue is Seedance 1.5 pro only.
- **Failed and expired tasks generally do not bill** — but tasks that succeed with garbage output DO bill. Always check the result URL before chaining further calls.

## Error reference (observed shapes)

| status / error | likely cause | fix |
|---|---|---|
| `403` / "model not activated" | account balance < ¥200 and no resource package | top up at the console, then retry |
| `400` "invalid model" | typo'd model name | use `doubao-seedance-2-0-260128` exactly (or the Fast variant) |
| `400` "invalid duration" | passed a string, or out of 4–15 range | integer in 4–15 |
| `400` "invalid content combination" | text+audio only / audio-only / mixed first_frame + reference_image | review the valid-combinations list above |
| `status: "failed"` with face-related message | a real human face slipped into the reference | use `asset://` portrait, or retry without the reference, or use a Seedance-generated face per the trust rules |
| `status: "expired"` | task queued/ran past `execution_expires_after` | retry, or raise the timeout (max 259200s = 72h) |
| `status: "succeeded"` but bad output | content moderation tripped silently / prompt was misread | check `r.revised_prompt` to see what the model actually saw; rewrite to be more concrete |
| `429` | hit RPM or concurrency | back off; concurrency limit (3 personal / 10 enterprise) bites first for batch workloads |
| URL returns 404/403 after a while | the result video URL expired (>24h) | re-fetch via TOS data subscription archive, or generate again |

## When to pick Seedance vs other Trove video modules

- **seedance (this module)** → multimodal reference is the killer feature (text + image + video + audio in one call). Best for "I want a video that combines THIS style image + THIS motion clip + THIS voice." Also strongest at video edit / extend.
- **minimax** → simpler text-to-video / image-to-video. Good when you only need T2V or I2V and want a different style bias (more anime / illustrative).
- **fal-ai** → marketplace of many video models including non-Bytedance ones. Use when you want to A/B several providers without integrating each.

Rule of thumb: known-good multimodal pipeline → seedance. Single-modality T2V / I2V → minimax or fal-ai depending on style.
